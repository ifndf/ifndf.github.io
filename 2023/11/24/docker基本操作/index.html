docker基本操作

发布于 2023-11-24 daily 

创建并运行：
docker container run nginx

停止运行：
docker container stop

查看状态：
docker container ls -a
docker container ps -a

批量操作：
docker container stop $(docker container ps -qa）
docker container rm $(docker container ps -qa）
不能直接删除正在运行的container,可以加上-f强制

指定端口运行：
docker container run -p 80:80 nginx （前台运行attach）
docker run -d -p 80:80 nginx (后台运行detech)
docker attach (连接回前台)

查看运行日志：
docker container logs -f

创建并进入交互模式：
docker container run -it ubuntu sh
docker container run -it busybox sh

后台运行后，进入交互：
docker exec -it sh

拉取指定版本得image:
www.hub.docker.com
docker image pull nginx:1.20.0

查看image具体信息：
docker image

先停止/删除容器，才能删除image。

image导出导入：
docker image save nginx:1.20. -o nginx.image
docker image load -i . nginx.image

使用dockerfile创建、提交
写好dockerfile,在当前路径下:
docker image build -t <username/imagename:version_tag> .

或者做好后再打tag:
docker image tag ori_image <username/imagename:version_tag>

docker login
docker image push <username/imagename:version_tag>
docker pull <username/imagename:version_tag>

运行container并时修改内容后，如何保存为新的image?

docker container commit ori_container_ID <username/imagename:version_tag>

示例：
docker container run -it ubuntu:21.04 sh
在sh中安装好python环境，编写好python脚本hello.py
然后commit为新image:
docker container commit ori_container_ID <username/imagename:version_tag>
重新运行新image:
docker container run -it xiaopeng163/python-demo python3 /hello.py

scratch是什么image?
示例：
编写一个C程序：
#include <stdio.h> int main() { printf("hello docker n"); }

gcc –static -o hello hello.c
生成hello

编写dockerfile:
FROM scratch
ADD hello
CMD [“/hello”]

docker build -t hello .

docker container run -it hello

显示image的layer:
docker image history hello
